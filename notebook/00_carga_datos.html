1. Dimesiones del data set¶

  • El dataset contiene 307,511 filas (observaciones o registros) y 122 columnas (variables).
  • Cada fila representa una persona o cliente, y las columnas son las caractersticas asociadas a cada uno.

2. Tipos de datos¶

  • int64: Variables numricas enteras. Por ejemplo, SK_ID_CURR parece ser un identificador nico de cliente y TARGET es el objetivo que estamos tratando de analizar.
  • object: Variables categricas o texto. Por ejemplo, NAME_CONTRACT_TYPE podra indicar el tipo de contrato y CODE_GENDER el gnero.
  • float64 (mostrado ms adelante): Variables numricas decimales, como las solicitudes de crdito (AMT_REQ_CREDIT_BUREAU_DAY).

3. Valores nulos¶

  • Columnas con 0 valores nulos: Estas variables estn completas, es decir, no tienen datos faltantes.
    • Ejemplo: SK_ID_CURR, TARGET, CODE_GENDER no tienen valores nulos.
  • Columnas con valores nulos (como AMT_REQ_CREDIT_BUREAU_YEAR):
    • Esta columna tiene 41,519 valores nulos, lo que representa una proporcin significativa del total.

Seleccin de threshold por filas y columnas para valores nulos

El anlisis de los valores nulos revela lo siguiente:

Por columnas:

Existen 122 columnas en el dataset, de las cuales varias tienen un porcentaje significativo de valores nulos.

Las columnas COMMONAREA_MED, COMMONAREA_AVG y COMMONAREA_MODE presentan cerca del 69.87% de valores faltantes, lo que las hace candidatas para ser eliminadas o imputadas segn su relevancia en el anlisis.

Algunas columnas, como NAME_HOUSING_TYPE y NAME_EDUCATION_TYPE, no presentan valores nulos y estn completas. Por filas:

Hay 307,511 filas en el dataset, y un nmero considerable tiene hasta un 50% de sus valores faltantes.

Esto podra indicar la necesidad de aplicar un threshold para eliminar estas filas, especialmente si contienen poca informacin relevante. Relevancia de TARGET en las filas con nulos:

Se incluy la variable TARGET para observar cmo se distribuyen los valores faltantes en relacin con la variable objetivo. Esto permitir evaluar si las filas con ms nulos tienen una relacin significativa con el incumplimiento (TARGET = 1).

Explicación: ¿Por qué se eligieron estas 4 variables?¶

  1. Relevancia en el contexto financiero:

    • Estas variables son indicadores importantes del perfil del cliente y su potencial comportamiento de pago.
    • Por ejemplo, variables como el gnero (CODE_GENDER) o la ocupacin (OCCUPATION_TYPE) pueden estar correlacionadas con el riesgo de incumplimiento (TARGET).

  2. Diversidad de informacin:

    • Cada una de estas variables ofrece una perspectiva nica:
      • CODE_GENDER: Informacin demogrfica.
      • NAME_CONTRACT_TYPE: Informacin sobre el tipo de producto financiero.
      • NAME_EDUCATION_TYPE: Informacin sobre el nivel educativo del cliente.
      • OCCUPATION_TYPE: Informacin sobre la estabilidad laboral y nivel de ingresos.

  3. Impacto potencial en los modelos:

    • Estas variables suelen ser incluidas en modelos de scoring crediticio porque ayudan a segmentar los clientes segn sus caractersticas clave.

  4. Facilidad de interpretacin:

    • Estas variables tienen pocas categoras (excepto OCCUPATION_TYPE), lo que facilita interpretar su impacto en el anlisis.
